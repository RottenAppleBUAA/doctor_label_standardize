# 医生标注标准化脚本使用说明

本文档说明如何使用新的医生标注标准化 pipeline：

- [`run_per_doctor_annotation_standardization_pipeline.sh`](./run_per_doctor_annotation_standardization_pipeline.sh)
- [`normalize_doctor_annotations_individually.py`](./normalize_doctor_annotations_individually.py)

这条新流程只做一件事：把医生标注 JSON 标准化到模型预测的标准牙位框上。

它不再生成原始叠加图，也不要求标注 JSON 数量必须是 2 个。

## 适用场景

适用于以下情况：

- 待处理数据路径希望自由指定
- 输出结果希望写在当前这批数据目录下面
- `annotations/` 下面可能有任意数量的医生标注 JSON
- 每个医生 JSON 需要分别标准化、分别输出

## 脚本行为

新 pipeline 的行为如下：

1. 扫描 `annotation-dir` 下所有 `.json` 文件
2. 对每个 JSON 文件单独读取其中的医生矩形框标注
3. 对每张图跑模型推理，得到标准牙位框
4. 将该医生的标注框逐个匹配到标准牙位框
5. 每个医生 JSON 单独输出一套标准化结果
6. 最后在总输出目录下生成一份批处理汇总

## 路径规则

这版脚本不依赖固定绝对路径。

支持传入绝对路径，也支持传入相对路径。

相对路径的解析规则是：

- 所有相对路径都以当前项目目录为基准
- 也就是脚本文件所在目录

涉及这个规则的参数包括：

- `--data-root`
- `--annotation-dir`
- `--image-dir`
- `--output-dir`
- `--config-file`
- `--weights`

## 必填参数

运行 shell pipeline 时，必须提供：

- `--data-root`
- `--image-dir`

其中：

- `--data-root` 表示当前这一批数据的根目录
- `--image-dir` 表示原图目录

如果不传 `--annotation-dir`，默认使用：

```text
<data-root>/annotations
```

如果不传 `--output-dir`，默认输出到：

```text
<data-root>/standardized_annotations_per_doctor
```

## 快速开始

在项目根目录执行：

```bash
cd SemiT-SAM
```

如果你的数据目录结构是：

```text
some_case/
  annotations/
    doctor_a.json
    doctor_b.json
    doctor_c.json
  images/
    001.png
    002.png
```

那么可以直接运行：

```bash
./run_per_doctor_annotation_standardization_pipeline.sh \
  --data-root ../some_case \
  --image-dir ../some_case/images
```

因为没有显式传 `--annotation-dir`，脚本会默认读取：

```text
../some_case/annotations
```

## 指定标注目录

如果医生标注 JSON 不在 `<data-root>/annotations` 下，可以显式指定：

```bash
./run_per_doctor_annotation_standardization_pipeline.sh \
  --data-root ../some_case \
  --annotation-dir ../some_case/my_annotations \
  --image-dir ../some_case/images
```

## 指定输出目录

如果不想使用默认输出目录，可以手动指定：

```bash
./run_per_doctor_annotation_standardization_pipeline.sh \
  --data-root ../some_case \
  --annotation-dir ../some_case/annotations \
  --image-dir ../some_case/images \
  --output-dir ../some_case/my_standardized_output
```

## 使用 CUDA

默认设备是 `cpu`。如果要改成 CUDA：

```bash
./run_per_doctor_annotation_standardization_pipeline.sh \
  --data-root ../some_case \
  --image-dir ../some_case/images \
  --device cuda
```

## 调整标准化阈值

shell pipeline 支持把额外参数透传给 Python 标准化脚本，例如：

```bash
./run_per_doctor_annotation_standardization_pipeline.sh \
  --data-root ../some_case \
  --image-dir ../some_case/images \
  --single-min-query-cover 0.30 \
  --single-min-match-score 0.40 \
  --max-box-width-percent 20 \
  --max-box-height-percent 26
```

这些参数会直接传给 [`normalize_doctor_annotations_individually.py`](./normalize_doctor_annotations_individually.py)。

## 直接运行 Python 脚本

如果你想绕过 shell pipeline，也可以直接运行：

```bash
conda run -n gd_env python normalize_doctor_annotations_individually.py \
  --data-root ../some_case \
  --annotation-dir ../some_case/annotations \
  --image-dir ../some_case/images
```

## 输出结构

默认输出目录如下：

```text
<data-root>/standardized_annotations_per_doctor/
```

目录下会为每个医生 JSON 生成一个单独子目录。例如：

```text
standardized_annotations_per_doctor/
  doctor_a/
    normalized_annotations.json
    ignored_annotations.json
    summary.json
  doctor_b/
    normalized_annotations.json
    ignored_annotations.json
    summary.json
  doctor_c/
    normalized_annotations.json
    ignored_annotations.json
    summary.json
  batch_summary.json
```

其中：

`normalized_annotations.json`：

- 当前这个医生 JSON 的标准化结果
- 每张图包含模型标准框和匹配上的医生标注

`ignored_annotations.json`：

- 当前这个医生 JSON 中未进入最终结果的框
- 常见原因包括 `box_too_wide`、`box_too_tall`、`multi_tooth_box`、`no_standard_overlap`、`low_single_match_quality`、`image_not_found`、`no_standard_boxes_predicted`

`summary.json`：

- 当前这个医生 JSON 的统计汇总
- 包含 `global`、`per_image`、`ignored_reason_counts`

`batch_summary.json`：

- 这次批处理的总清单
- 记录处理了哪些 JSON、每个 JSON 的输出目录在哪里

## 文件命名规则

每个医生 JSON 的输出子目录名默认来自该 JSON 的文件名。

例如：

- `doctor_a.json` -> `doctor_a/`
- `prelabels_001.json` -> `prelabels_001/`

如果多个文件清洗后目录名冲突，脚本会自动追加后缀，例如：

- `doctor/`
- `doctor_2/`

## 运行前提

运行前需要满足：

- `conda` 可用
- 目标环境里已安装 `torch`、`opencv-python` 等依赖
- 模型权重文件存在
- 配置文件存在
- JSON 中的 `fileName` 能在 `image-dir` 下找到对应原图

默认相对路径下会使用：

- 配置文件：`configs/coco/instance-segmentation/maskformer2_R50_bs16_50ep.yaml`
- 权重文件：`pretrained_models/SemiTNet_Tooth_Instance_Segmentation_32Classes.pth`

这两个默认值也是按项目目录解析的，不是固定绝对路径。

## 常见问题

`No annotation json files found ...`：

- `annotation-dir` 目录下没有任何 `.json` 文件

`Unable to read image ...`：

- 原图缺失，或者 JSON 里的 `fileName` 和实际文件名对不上

`image_not_found`：

- 脚本没有崩溃，而是某张图在 `image-dir` 下找不到，会写入 `ignored_annotations.json`

`no_standard_boxes_predicted`：

- 模型在某张图上没有预测出标准牙位框，也会写入 `ignored_annotations.json`

CUDA 初始化失败：

- 使用 `--device cuda` 时，脚本会先做一次 CUDA 预热
- 如果仍失败，优先检查当前环境中的 `torch` 和 CUDA 是否匹配
